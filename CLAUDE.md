## Project Overview

This is a Rust project that renders curl noise particle animations on RGB LED matrix displays. It targets Raspberry Pi hardware with LED matrix panels (typically 64x64 pixel displays). The project uses a 3D simplex noise implementation with derivative calculation to generate a divergence-free 2D vector field (curl noise) that drives particle motion.

## Build and Run

This project cross-compiles from a development machine to `aarch64-unknown-linux-gnu` (ARM64) for the Raspberry Pi target.

**Build:**
```bash
cargo build --release
```

**Run examples:**
```bash
cargo run --release --example image
cargo run --release --bin curl_matrix -- --particle-count 2000 --field-scale 0.05
```

The main binary accepts command-line arguments:
- `--particle-count`: Total number of particles (default: 2000)
- `--field-scale`: Scaling of the vector field (default: 0.05)
- `--particle-speed`: Scaling of particle movements (default: 0.01)
- `--flow-speed`: Scaling of the overall flow (default: 0.09)
- `--particle-ttl`: Maximum particle lifetime (default: 2500)

Standard `rpi-led-matrix` arguments are also supported (hardware configuration).

## Architecture

**Main Components:**

- `src/main.rs` - Entry point, particle system simulation loop, and canvas rendering. Creates a pre-computed vector field using curl noise, then spawns particles that follow the field. Particles wrap around screen edges and regenerate when their TTL expires.

- `src/simplex.rs` - 3D simplex noise implementation with derivative calculation (based on Stefan Gustavson's public domain algorithm). The `with_derivatives_3d()` function returns both the noise value and its gradient vector, which is used to compute the curl.

- `examples/image.rs` - Simple example showing how to display a static RGB image on the matrix using the `embedded-graphics` crate.

**Key Dependencies:**
- `rpi-led-matrix` - Rust bindings for the rpi-rgb-led-matrix C++ library (local path dependency)
- `nalgebra` - Linear algebra for vector operations (`Point2`, `Point3`, `Vector2`, `Vector3`)
- `clap` - Command-line argument parsing
- `rand` - Random number generation for particles

**Curl Noise Algorithm:**

The vector field is generated by computing the curl of a 3D simplex noise potential field:
1. `simplex::with_derivatives_3d()` computes noise value and partial derivatives at (x, y, time)
2. `curl_2d()` converts the potential field gradient to a divergence-free vector field: `(dy, -dx)`
3. The result is a smooth, incompressible flow field that particles follow

## Cross-Compilation Setup

Build configuration in `.cargo/config.toml`:
- Target: `aarch64-unknown-linux-gnu`
- Linker: `/usr/bin/aarch64-linux-gnu-gcc`

The `rpi-led-matrix` dependency is a local path reference to `rust-rpi-rgb-led-matrix/rpi-led-matrix` and must be available for compilation.
